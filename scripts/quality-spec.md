# type-globe 問題データ品質基準 (round-1 rebuild)

旧データは下記のような品質問題が多く、全件 rebuild する。

## 旧データの典型的な問題 (避けるべき例)

- **漢字の誤り**: 「織田栄一郎」(誤) ← 正しくは「尾田栄一郎」(ONE PIECE 作者)
- **公式表記の無視**: 「庵野ひであき」(誤) ← 正しくは「庵野秀明」/「畑ヶ樫」(誤) ← 正しくは「はたけカカシ」
- **架空人物の混入**: 「根里俊」のような実在しない名前
- **不自然な日本語**: 「死の帳から力を得る少年」(誤) ← 「死神のノートから力を得る少年」
- **テンプレ量産選択肢**: 「古代エジプト人 / 古代ローマ人 / 古代ギリシャ人 / 古代メソポタミア人」のような無思考な羅列
- **英訳ズレ**: ja と en で別のことを聞いている、訳が不正確
- **読みの不統一**: 同じ文中で漢字の開きが揃っていない
- **長音の崩れ**: `ボール` を `boru` で登録 (本来 `bo-ru` 必須)
- **ジャンル誤認**: デスノートを「心理スリラー」(誤、サスペンス)

## 品質ルール (全件必達)

### 1. 事実の正確性
- 教科書的・百科事典的に確認できる事実のみ
- 固有名詞 (人名・地名・作品名) の漢字を必ず検証
- 不確かなら WebSearch で確認してから書く
- マイナー過ぎる雑学は避ける

### 2. 自然な日本語
- ネイティブが読んで自然な問題文
- 漢字とカタカナの混ぜ方が不自然なものを避ける
- 漢字を強引にひらがなに開かない、開くなら全体方針を統一

### 3. 二言語アライメント
- `question_text.ja` と `question_text.en` は完全に同じ意味
- `choice.ja` と `choice.en` は同じ概念
- 両方の言語でそれぞれ自然に読める表現 (逐語訳禁止)

### 4. 妥当な distractor
- 誤答候補も同じジャンル・近い概念で「迷いそう」なものに
- テンプレ羅列 (古代X / 古代Y / ... など) 禁止
- 同じ問題内で 4 つの選択肢は明確に異なる事物

### 5. ja_typings 形式 (厳格)
- 小文字 ASCII のみ
- ヘボン式 (`shi` / `chi` / `tsu` / `fu` / `ji`)、kunrei 不可
- カタカナ長音 `ー` → ハイフン `-`
  - 例: `サーバー` → `sa-ba-`、`ボール` → `bo-ru`、`ヒーロー` → `hi-ro-`
- ン+母音/y は `nn` 二重で登録
  - 例: `観音` → `kannon`、`翻訳` → `honnyaku`
- ン+子音 / 末尾 は `n` 単数
  - 例: `ドラゴン` → `doragon`、`東京` → `tokyo`
- ひらがな長音 `おう/おお` は両方登録
  - 例: `東京` → `["tokyo", "toukyou"]`、`大阪` → `["osaka", "oosaka"]`
- 漢字読みの揺れがあれば全て登録
  - 例: `日本` → `["nihon", "nippon"]`、`七人` → `["shichinin", "nananin"]`
- 英字・数字はそのまま小文字

### 6. prefix 衝突なし
- 同じ問題内で、ある選択肢の `ja_typings` が別の選択肢の prefix になってはいけない
- 例: `["tokyo"]` と `["tokyoshi"]` を同じ問題に並べない

### 7. 重複なし
- 同じ事実を問う問題を作らない
- 4 選択肢の組み合わせが過去と被らない
- パターン (問い方) を変える

### 8. メタ
- `ja_reviewed: true` を必ず立てる
- `image_path: null`
- `id` は指定された範囲・桁数で連番

## スキーマ

```json
{
  "id": "q0001",
  "genre": "science",
  "question_text": {"ja": "...", "en": "..."},
  "choices": [
    {"ja": "...", "en": "...", "ja_typings": ["..."]},
    {"ja": "...", "en": "...", "ja_typings": ["..."]},
    {"ja": "...", "en": "...", "ja_typings": ["..."]},
    {"ja": "...", "en": "...", "ja_typings": ["..."]}
  ],
  "correct_answer_index": 0,
  "image_path": null,
  "ja_reviewed": true
}
```

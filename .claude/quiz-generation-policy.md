## 問題作成の重要ルール

### 1. 入力仕様（最重要）

#### 日本語の選択肢
- **ひらがな、カタカナ、アルファベットのみ使用**
- **漢字は絶対に使わない**
- 理由: ユーザーはフリック入力やキーボードでタイピングする。漢字変換は不要。

**正しい例:**
- ✓ `とうきょう`
- ✓ `トウキョウ`
- ✓ `Tokyo`

**間違った例:**
- ✗ `東京` （漢字は使用禁止）

#### 英語の選択肢
- アルファベット、数字、記号のみ使用

#### 選択肢の長さ
- 各選択肢は20文字以内を推奨
- タイピングしやすさを最優先

### 2. オリジナリティ

既存のクイズサイトにありがちな問題ではなく、独自性のある問題を作成:

- 単純な定義問題ばかりではなく、応用的・実践的な問題を含める
- 複数の知識を組み合わせた問題を作る
- ユニークな切り口や視点を持つ問題を含める
- 最新のトレンドや話題も取り入れる
- トリビア的な面白い知識も織り交ぜる

### 3. 難易度

- 初級から上級まで混在させる
- プログラマー・アニメ/ゲームファン向けにやや偏重
- 一般常識や高校レベルの学問も含む

### 4. JSON形式

```json
{
  "id": "q00001",
  "genre": "programming",
  "question_text": {
    "ja": "日本語の問題文",
    "en": "English question text"
  },
  "choices": [
    {"ja": "せんたくし1", "en": "Choice 1"},
    {"ja": "せんたくし2", "en": "Choice 2"},
    {"ja": "せんたくし3", "en": "Choice 3"},
    {"ja": "せんたくし4", "en": "Choice 4"}
  ],
  "correct_answer_index": 0,
  "image_path": null
}
```

### 5. ID命名規則

- フォーマット: `q` + 5桁の連番（ゼロパディング）
- 例: `q00001`, `q00002`, ..., `q10000`

## 品質管理

### チェックポイント
1. **漢字チェック**: 日本語の選択肢に漢字が含まれていないか
2. **長さチェック**: 選択肢が20文字以内か
3. **重複チェック**: 同じような問題が複数ないか
4. **バランスチェック**: ジャンルの配分が適切か
5. **正解位置**: 正解がランダムに配置されているか（特定の位置に偏らない）

### 定期レビュー
- 100問ごとに内容をレビュー
- ユーザーからのフィードバックを反映
- トレンドに合わせて問題を更新
